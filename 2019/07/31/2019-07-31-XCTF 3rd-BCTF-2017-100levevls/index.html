100levevls

程序开启了NX、PIE，考察的是PIE的Bypass手段，之前我总结过，发在了安全客，传送门

大致看一下程序的功能，有个菜单：

选项1中首先输入两个数字，然后将这两个数字相加和99相比，来确定后面递归计算的次数

选项2中判断bss段上unk_20208C变量是否是0，若不是0的话则puts出来system的地址，否则则puts出来字符串NO PWN NO FUN

通过分析测试发现在problem函数中存在栈溢出漏洞，并且溢出的长度足够覆盖到ret地址
在hint函数中，会把system地址写到ebp-110处，在go函数处，ebp的值和hint函数一样，v5所在的地址也是ebp-110，也就是system的地址，并且当v2小于0的时候是不会修改ebp-110中的值

栈溢出实在go函数中的problem函数，那么problem函数的栈帧就是在go函数的esp上面，我们可以将problem函数ret的时候滑到ebp-110，也就是system函数地址上，但是没有字符串”/bin/sh”，我们可以借用v6 = v5 + v3;将修改成one_gg,然后直接滑到one_gg上就可以直接获取到shell

那么用什么来滑呢，既然程序开启了PIE，那么我们就借用syscall(0xffffffffff600000)，用它的原因就是它的地址是固定不变的

由于problem函数使用了递归函数，那么我们构造栈溢出必须在最后一次ret时才可以

problem函数退出时栈的分布时这样的，我们可以把syscall地址填充到0x7fffffffdd28、0x7fffffffdd30、0x7fffffffdd38，这样的话就会滑到我们one_gg的地址上了
