one_string

没开任何保护，拖入IDA中查看发现这道题是静态编译的pwn

通过字符串找到关键函数

该程序就一个输入，也就是最开始的You know all, Please input:
选项一是create,先是读入一个需要malloc的size，然后读入size个字符

edit选项：先是读入要修改的index，然后读入size个字符，用chunk中存的字符串的长度来修改原先的size的值（漏洞点就在这里）

delete选项，读入index，free对应的chunk

攻击流程：

  • 通过edit选项可触发off-by-one
  • 然后构造chunk来触发unlink，控制heap_list
  • 然后在bss段上写上shellcode，将free_hook中写上shellcode的地址
  • 调用delete选项，触发shellcode，得到shell

exp:

需要注意的是构造unlink时需要将target-0xc的值刚好等于0x80eba40，因为在unlink之前需要edit前一个chunk，最大的覆写size是5，所以要这样构造

但是题目在服务器上部署的时候是不一样的：

所以我们要先把我们的payload进行base64加密之后再发送到服务器上面去，然后getshell
