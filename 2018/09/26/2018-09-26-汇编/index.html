汇编语言随书笔记

二进制的开始

简介

在我看来，汇编语言是一个相对低级的语言，它是二进制上一层的语言，并不像C\C++\PYTHON一样那么简单、易懂，学习汇编是需要一些计算机硬件上的知识。

我是根据《汇编语言》–王爽 这本书来展开学习的。

第一章

主要是讲解了计算机的一些组成元件以及工作原理，并且系统的讲到汇编语言，更偏向于计算机基础性的知识。

  • 机器语言是机器指令的集合
  • 汇编语言的主题是汇编指令，汇编指令时机器指令便于记忆的书写格式
  • 一个存储器有128个存储单元，它可以存储128个Byte
  • CPU要从内存中读取数据，首先要制定存储单元的地址，其次是器件的选择，读或写的命令(控制信息)，最后读或写的数据
  • CPU有专门和其他芯片的导线，成为总线。总线分为地址总线、控制总线、数据总线
  • 一个CPU有N根导线，则可以说这个CPU的地址总线的宽度为N。这样的CPU最多可以寻找2的N次方的内存单元
  • CPU和内存或者是其他器件的数据传送是通过数据总线来进行的，数据总线的宽度决定了CPU和外界传送的速度
  • CPU对外部器件的控制是通过控制总线来进行的，有多少根控制总线，就意味着CPU对外部器件的多少种控制
  • CPU可以直接使用的信息在存储器中存放

第二章

主要介绍了CPU中寄存器的工作原理以及引入段的概念

  • 通用寄存器：AX、BX、CX、DX，这些寄存器通常来存放一些一般性的数据
  • AX分为AH(高8位)、AL(低8位)
  • 字节：byte,一个字节是由8个bit组成
  • 字： word，一个字由两个字节组成，两个字节分别称为这个字的高位节和低位节。
  • mov X,Y :将Y中的指送入到X中，（X=Y）
  • add X,Y :将X中的指加上Y送入X中，（X=X+Y）
  • 在进行数据传送或运算时，要注意指令的两个操作对象的位数应当是一致的
  • 物理地址 = 段地址*16 + 偏移地址
  • 段寄存器：CS、DS、SS、ES
  • CS:IP 指向的是CPU当前要读取指令的地址，CS中存放着段地址，IP中存放着偏移地址
  • 在8086机中，任意时刻，CPU将CS:IP指向的内容当作指令执行
  • jmp X:Y 将X的值送到CS中，将Y的值送到IP中
  • jmp X 将X的值送到IP中

第三章

主要介绍了内存访问、数据传送以及引入栈的概念

  • 在CPU中，用16位寄存器来存储一个字，高8位存放高字节，低8位存放低字节
  • 内存单元式字节单元(8位)
  • 字单元：存放一个字型数据(16位)的内存单元，由两个地址连续的内存单元组成。高地址内存单元存放字型数据的高位字节，低地址内存单元存放字型数据的低位字节
  • 将起始地址为N的子单元简称为N地址单元
  • DS:[address] DS为段寄存器，存放数据的段地址
  • 设置DS的值：1. mov bx,1000H 2. mov ds,bx
  • sub X,Y : 将X的值减去Y的值再送到X中，（X=X-Y）
  • 栈：是一种具有特殊的访问方式的存储空间。特殊性：先进后出原则
  • SS:SP : 任意时刻，这个指向栈顶元素
  • push ax : I.SP = SP -2 II.将AX中的数据放入栈中
  • pop ax : I.将指向的指送到AX中 II. SP = SP + 2
  • 数据出栈后，原来地址上的数据并不消失，而还是原来的指，直到再往里面写进去数据将其覆盖。如果不向栈中写入其他的数据，那么之前的数据不会消失，只是不在栈中
  • 栈段的长度小于或等于64KB

第四章

主要讲解了一个汇编源程序到一个windows可执行程序(.exe)的过程。这一章我没有细看，所以在这里没有什么总结。

第五章

介绍了一个新的指令：loop，阐述了[bx]的另一层意思，还有引进了段前缀

  • [bx] 表示一个内存单元，偏移地址在bx中，段地址默认在DS中
  • LOOP：为循环指令，循环次数在CX中存放
  • loop 标号 ：当CPU执行loop指令的时候，第一，(cx) = (cx) -1;第二,判断cx中的值，不为零则转至标号处执行程序，如果为零则向下执行
  • 标号代表着一个地址
  • 段前缀：mov ax,cs[0] mov ax,ss:[0] 这里面cs、ss称为段前缀

第六章

本章将数据段和代码段、栈端联系到了一起。引入包含多个段的程序

  • dw：定义字型数据(每一个字占两个字节)
  • 定义多个段的方法：就是和定义代码段的方法是一样的，只是不同的段，段名字不一样
  • 在一个程序中，短命就相当于一个标号，代表了段地址

第七章

本章更详细的讲解了寻找内存地址的方法

  • and：可以将操作对象的相应位设置为0，其他位不变
  • or ：可以将操作对象的相应位设置为1，其他位不变
  • db ：以字节型定义数据
  • idata：称为立即数，直接包含在机器指令中的数据
  • [bx+idata]:可以用来进行数组的处理
  • si、di：是8086CPU中和bx功能相近的寄存器，但是不能狗分成两个8位的寄存器
  • 扩展寻址方式：[bx+si]、[bx+di]、[bx+si+idata]、[bx+di+idata]
  • 一般来说，在需要暂存数据的时候，我们都应该使用栈
  • 数据入栈的时候是先占据高地址

第八章

详细的阐述了计算机是怎么处理数据的

  • reg表示寄存器，sreg表示段寄存器
  • 在8086CPU中只有bx、si、di、bp可以用在[…]中来进行内存单元的寻址
  • 在[…]中，这4个寄存器可以单个出现，或只能以:bx&si、BX&di、bp&si、bp&di
  • 只要在[…]中使用寄存器bp，而指令中没有显性的给出端地址，段地址默认在ss中
  • 段地址：SA　偏移地址：EA
  • 通过[bx+idata+si]来访问结构体中的数据
  • dup：是和dd、dw、db等数据定义伪指令配合使用的，用来进行数据的重复；如db 0 dup (8)用来定义八个字节的0，相当于db 0,0,0,0,0,0,0,0

寻址方式：

在计算机中，识别指令处理的数据的长度：

  1. 通过寄存器指明要处理的数据的尺寸
  2. 在没有寄存器名存在的情况下，用操作符X ptr指明内存单元的长度，X在汇编指令中可以为word、byte
  3. word ptr指明了指令访问的内存单元式一个字单元
  4. byte ptr指明了指令访问的内存单元式一个字节单元
  5. 除此之外，push、pop指令默认的是进行字操作

div指令

div指令是除法指令，规则如下：

  • 除数：有8位和16位，在寄存器或者内存单元中
  • 被除数：默认放在AX或者DX和AX中，如果除数是8位，则被除数是16位，默认在AX中存放；如果除数为16位，则被除数位32位，在DX和AX中存放，DX存高16位，AX存低16位
  • 结果：如果除数是8位，则AL存储除法操作的商，AH存储除法操作的余数；如果除数是16位，则AX中存储除法操作的商，DX中存储除法操作的余数
  • dd：是用来定义双字型数据的

第九章

引入了新的指令：转移指令，在程序中，难免会碰到一些转移，如if…else…,switch…case…,都包含这转移指令

转移指令的定义：可以修改IP，或者同时修改CS和IP的指令；通俗的来讲，转移指令就是可以控制CPU执行内存中某处的代码的指令

  • 只修改IP，称为段内转移，比如:jmp ax
  • 同时修改CS和IP时，称段间转移，比如jmp 1000:0
  • 段内转移分为短转移和近转移
  • offset：取得标号的偏移地址
  • 有时候CPU在执行jmp指令时并不需要转移的目的地址
  • 依据位移进行转移的jmp指令
    • jmp short 标号(转到标号处执行指令)，这种格式是实现段内短转移，它对IP的修改范围为-128~127
      • 上一条命令其实平不是给出来标号的段地址和偏移地址，而是给出来将要转移的移位，(IP) = (IP) + 8位位移
      • 8位位移 = 标号处的地址-jmp指令后的第一个字节的地址
    • jmp near ptr 标号：它实现的是段内近转移
      • (IP) = (IP) + 16位位移
      • 16位位移 = 标号处的地址-jmp指令后的第一个字节的地址
  • 转移的目的地址在指令中的jmp指令
    • jmp far ptr 标号：实现的是段间转移，又称为远转移
      • (CS) = 标号所在段的段地址；(IP) = 标号所在段的偏移地址
  • 转移地址在寄存器中的jmp指令
    • jmp 16位reg
      • (IP) =(16位reg)
  • 转移地址在内存中的jmp指令
    • jmp word ptr 内存单元地址(段内转移) ：从内存单元地址处开始存放着一个字，是转移目的的偏移地址
    • jmp dword ptr 内存单元地址(段间转移)：从内存单元地址处开始存放着两个字，高地址处的字是转移目的的段地址，低地址处的字是转移目的的偏移地址
  • jcxz：该指令时有条件转移指令，所有的有条件转移指令都是短转移，在对应的机器码中包含转移的位移，对IP的修改范围都为-128~127
    • jcxz 标号 (如果(cx)=0,转移到标号处执行)
    • 当(cx)=0,(IP) = (IP) + 8位位移
    • 类C表示方法：if((cx)==0) jmp short 标号
  • loop: 该指令在之前就已经学习过，是循环指令，在这个补充说明到：所有的循环指令都是短转移，在对应的机器码中包含转移的位移
    • (cx) = (cx) - 1
    • 如果(cx) = 0，(IP) = (IP) + 8位位移

第十章

这一章描述了在一个程序中，是怎么调用一个函数并且返回被调用的返回值给调用的函数

  • call指令：
    • 依据位移进行转移的call指令
      • call 标号(将当前的IP压入栈中，转到标号处执行命令)
      • 相当于执行：push IP；jmp near ptr 标号
    • 依据目的地址在指令中的call指令
      • call far ptr 标号：实现的是段间转移
      • 相当于执行：push CS; push IP; jmp far ptr
    • 转移地址在寄存器中的call指令
      • call 16位reg
      • 相当于执行：push IP; jmp 16位reg
    • 转移地址在内存中的call指令
      • call word ptr 内存单元地址
        • 相当于执行push IP；jmp word ptr 内存单元地址
      • call dword ptr 内存单元地址
        • push CS；push IP；jmp dword ptr 内存单元地址
  • ret：用栈的数据修改IP的内容，实现进转移；(pop IP)
  • retf：用栈的数据修改CS、IP的内容，实现远转移；(pop IP、POP CS)
  • 用call和ret指令来完成程序对函数的调用
  • mul 指令：是乘法指令
    • 两个想成的数，要么都是8位，要么都是16位。如果是8位，一个默认放在al中，另一个放在8位的reg或者内存字节单元中；如果是16位，一个默认在AX中存放，另一个放在16位reg或者内存字单元中
    • 结果：如果是8位乘法，结果默认放在AX中；如果是16位乘法，结果高位默认放在DX中，地位在AX中放

调用函数时，参数的传递方法：

  • 用寄存器来存储参数和结果是最常见的方法。调用者和子程序的读写操作恰恰相反：调用者将参数送入参数寄存器中，从结果寄存器中取出；子程序从参数寄存器中取到参数，将返回值送入结果寄存器，但是这种方法是有缺陷的，如果参数多的话，则就会发生寄存器冲突的问题
  • 当参数多的时候，在子程序的开始将子程序中所用到的寄存器中的内容都保存起来，在子程序中返回前再回复。可以用栈来保存寄存器中的内容
  • 在这里因为牵扯到了栈的使用，所以要特别注意寄存器入栈和出栈的顺序

第十一章

本章介绍了在8086CPU中的标志寄存器，以及跳转指令

  • 标志寄存器的作用：
    • 用来储存相关指令的某些执行结果
    • 用来为CPU指令执行相关指令提供行为依据
    • 用来控制CPU的相关工作方式
  • flag(标志)寄存器的结构：
  • ZF标志：
    • 零标志位，他记录相关指令执行后其结果是否为0
    • 如果结果为0，则zf = 1，果如结果不为0，那么zf=0
  • PF标志：
    • 奇偶标志位，他记录了相关指令执行后，其结果的所有bit位中1的个数是否为偶数，如果1的个数为偶数，则zf=1，如果为奇数，zf=0
  • SF标志：
    • 符号标志位，他记录了相关指令执行后，其结果是否为负，如果为负，sf=1，如果非负，sf=0
    • 在计算机中通常用补码来表示有符号数据
  • CF标志：
    • 进位标志位，在进行无符号数运算的时候，他记录了运算结果的最高有效位向更高位的进位值，或者从更高位的借位值
  • OF标志：
    • 溢出标志位，他记录了有符号数运算的结果是否发生了溢出，如果发生溢出，OF= 1，若没有，OF= 0
  • DF标志
    • 方向标志位，在串处理指令中，控制每次操作后si,di的增减
    • df=0 每次操作后的si，di递增
    • df=1 每次操作后的si，di递减
    • cld 指令：将df的值设置为0
    • std 指令：将df的值设置为1

adc指令

  • 这个指令时带进位加法指令，他利用了CF位上记录的进位指
  • 指令格式 : adc 操作对象1，操作对象2
  • 功能：操作对象1 = 操作对象1 + 操作对象2 + CF
  • adc ax,bx —–> (ax) = (ax) + (bx) + CF
  • 需要注意的是，inc和loop指令不影响CF位

sbb指令

  • 这个指令时带借位减法指令
  • 指令格式 : sbb 操作对象1，操作对象2
  • 功能：操作对象1 = 操作对象1 - 操作对象2 - CF
  • sbb ax,bx —–> (ax) = (ax) - (bx) - CF

cmp指令

  • cmp指令时比较指令，它的功能相当于减法指令，只是不保存结果。cmp指令执行后，将对标志寄存器产生影响

movsb

  • 串传送指令,将ds:si指向的内存单元的字节送入es:di中，然后根据标志寄存器df位的值，将si和di递增或递减
  • 指令步骤

movsw

  • 串传送指令,将ds:si指向的内存单元的字送入es:di中，然后根据标志寄存器df位的值，将si和di递增或递减
  • 指令步骤

rep

  • 作用是根据cx的值，重复执行后面的串传送指令

rep和movsb或者movsw的结合使用

  • rep movsw/movsb，传送(cx)个字符

检测比较结果的条件转移指令

  • 转移指的是它能够修改IP，而条件指的是他可以根据某种条件，决定是否修改IP
  • 所有条件转移指令的转移位移都是[-128,127]
指令 含义 检测的相关的标志位
je 等于则转移 zf=1
jne 不等于则转移 zf=0
jb 低于则转移 cf=1
jnb 不低于则转移 cf=0
ja 高于则转移 cf=0且zf=0
jna 不高于则转移 cf=1或zf=1

pushf和popf

  • pushf是将标志寄存器的值压栈
  • popf是从栈中弹出数据，送入标志寄存器
